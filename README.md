# Audio Watermarking System

Система цифровых аудиоводяных знаков на основе адаптивной эквализации для защиты авторских прав на аудиоконтент.

Выпускная квалификационная работа — МИРЭА, 2026. Автор: Лисин Сергей Вячеславович.

---

## О проекте

Система встраивает невидимый (неслышимый) идентификатор правообладателя непосредственно в структуру аудиосигнала в ультразвуковом диапазоне 20–24 кГц. Знак сохраняется после транскодирования в MP3 192+ кбит/с — стандартный профиль качества Яндекс Музыки и ВКонтакте Музыки — и позволяет сформировать верифицируемое техническое доказательство авторства.

**Ключевые характеристики:**

| Метрика | Результат |
|---|---|
| SNR (незаметность) | 24.8 – 31.2 дБ |
| Корреляция Пирсона | 0.82 – 0.96 |
| BER при MP3 ≥ 192 кбит/с | < 0.05 |
| Устойчивость к атакам | 7 из 10 типов |

---

## Как работает

1. User ID правообладателя прогоняется через SHA-256 → первые 32 бита формируют уникальный паттерн
2. Паттерн кодируется в синусоиды в диапазоне 20–21 кГц методом FSK или DSSS
3. Сигнал накладывается на оригинальный трек с амплитудой 1% и повторяется по всей длине файла
4. При детектировании система вычисляет корреляцию Пирсона между извлечённым и эталонным паттерном — если > 0.6, авторство подтверждено

---

## Методы встраивания

**FSK (Frequency Shift Keying)**
- Бит `0` → тон 20 000 Гц
- Бит `1` → тон 20 500 Гц
- Длительность символа: 100 мс
- Окно Ханна сглаживает границы символов

**DSSS (Direct Sequence Spread Spectrum)**
- Энергия знака распределяется по широкой полосе через псевдослучайную PN-последовательность
- Более устойчив к шумовым атакам
- Принцип аналогичен GPS и CDMA

## Запуск

### GUI-приложение

```bash
python gui_watermark.py
```

Четыре вкладки:
- **Встраивание** — выбор файла, ввод User ID, выбор метода (FSK / DSSS), спектрограммы до/после
- **Детектирование** — верификационный или слепой режим, цветовой результат, SNR и корреляция
- **Настройки** — частотный диапазон, амплитуда, длина паттерна
- **Справка** — описание метрик и руководство пользователя

### Использование как библиотека

```python
from watermark_system import AudioWatermark, WatermarkConfig

config = WatermarkConfig(
    freq_range=(20000, 21000),
    amplitude=0.01,
    duration_ms=100,
    pattern_length=32,
    method='fsk'
)

wm = AudioWatermark(config)

# Встраивание
wm.embed_watermark(
    input_path='track.wav',
    output_path='track_watermarked.wav',
    user_id='ivan123'
)

# Детектирование (верификационный режим)
result = wm.detect_watermark(
    input_path='track_watermarked.wav',
    user_id='ivan123'
)

print(result['detected'])      # True / False
print(result['correlation'])   # 0.89
print(result['snr_db'])        # 28.3
```

### Слепой режим детектирования

```python
# User ID не нужен — система ищет любой водяной знак
result = wm.detect_watermark(
    input_path='suspicious_file.mp3'
)

print(result['detected'])     # True
print(result['pattern'])      # 32-битный отпечаток для сравнения с реестром
```

## Поддерживаемые форматы

| Формат | Встраивание | Детектирование |
|---|---|---|
| WAV (16/24/32 бит) | ✓ | ✓ |
| FLAC | ✓ | ✓ |
| MP3 | — | ✓ |
| OGG | — | ✓ |
| M4A / AAC | — | ✓ |

> Встраивание выполняется только в форматы без потерь. Для файлов MP3/OGG система автоматически снижает целевой диапазон до 16 000–17 000 Гц.

## Устойчивость к атакам

| Атака | FSK BER | DSSS BER | Статус |
|---|---|---|---|
| Без атаки (WAV→WAV) | 0.02 | 0.01 | ✓ |
| MP3 320 кбит/с | 0.03 | 0.02 | ✓ |
| MP3 192 кбит/с | 0.04 | 0.03 | ✓ |
| Нормализация громкости | 0.03 | 0.02 | ✓ |
| Компрессия 2:1 | 0.05 | 0.03 | ✓ |
| Обрезка начала 10% | 0.05 | 0.04 | ✓ |
| FLAC→WAV | 0.02 | 0.01 | ✓ |
| Шум (SNR 20 дБ) | 0.06 | 0.04 | ⚠ |
| MP3 128 кбит/с | 0.08 | 0.06 | ⚠ |
| Питч-шифтинг +1 пт | 0.18 | 0.12 | ✗ |

✓ BER < 0.05 — норма · ⚠ граница · ✗ предел
